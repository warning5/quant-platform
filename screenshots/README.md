## 截图规范

### 分辨率与格式
- **分辨率**：1920×1080（全高清），保证清晰度
- **格式**：PNG（无损，适合含文字的界面截图）
- **命名**：小写 + 连字符，按上表文件名，**不要用中文或空格**

### 截图前准备
1. 启动后端：`java -jar backend/target/backend-1.0.0.jar`
2. 启动前端：`cd frontend && npm start`
3. 浏览器访问 `http://localhost:3000`
4. **数据准备**：确保有真实数据。如果是空库，先跑一次数据采集和因子计算
5. **浏览器**：使用 Chrome，缩放比例 100%（不要用 80% 或 125%）
6. **隐藏敏感信息**：如果界面有真实持仓金额等，可模糊处理

### 截图技巧
- 用浏览器全屏模式（F11）截，避免地址栏干扰
- 截图范围以"内容主体"为准，不需要浏览器边框
- 如果页面很长，截首屏即可（不要拼接长图，README 展示会变形）
- 图表类页面优先截"有数据的"，不要截空状态

## 如何截图（Windows）

推荐用 **Snipaste**（免费）或系统自带 **Win + Shift + S**：

```
1. 打开目标页面
2. 按 Win + Shift + S（或 Snipaste F1）
3. 框选内容区域
4. 保存为 PNG 到本目录
5. 按上表命名
```

## README 引用方式

截图完成后，README 会自动通过以下语法引用（已在 README 中预置）：

```markdown
![Dashboard](screenshots/01-dashboard.png)
```

如果某张截图暂时没有，README 会显示破图占位——**这是正常的**，等截图补上后自动恢复。

## 截图更新

平台大版本更新后（如 UI 改版、新增页面），需要重新截图替换。建议每次 Release 前检查截图是否过期。
