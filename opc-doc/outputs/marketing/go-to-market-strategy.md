# Quant Platform 推广策略：从 0 到 1000 种子用户

> 制定日期：2026-07-20
> 约束条件：个人独立开发、零预算、目标受众为量化学习者、商业化获客导向
> 核心信条：金融工具的推广，信任 > 流量。我们不买流量，我们买信任。

---

## 一、现状诊断：三个必须正视的矛盾

### 矛盾 1：商业化目标 vs 学习者受众（付费意愿最低）

量化学习者的典型画像：在校生、刚入行的量化新人、跨行转量化的程序员。他们的特征是：
- **时间多、钱少**：愿意花 3 小时跑通一个开源项目，不愿意花 99 元买 SaaS 订阅
- **白嫖心态重**：开源项目对他们来说是"免费教材"，不是"工具"
- **付费触发点不同**：他们不为"更好的数据源"付费，但可能为"不用自己部署""有人教""有社群"付费

**这不是死局**，但意味着你不能用"免费→付费增值"的经典 SaaS 漏斗。你需要一条**"学习者→传播者→未来付费者"**的长线漏斗。

### 矛盾 2：零预算 vs 金融工具的高信任成本

普通工具推广可以靠投流、裂变、KOL 推荐。但量化工具不行：
- 用户不会因为一篇软文就把钱交给一个没听过的平台
- 金融领域合规红线多（不能承诺收益、不能荐股、不能用极限词）
- 竞品已有先发优势：聚宽、米筐、优矿、掘金量化占据了"平台型"心智

**零预算下的解法**：用技术深度内容建立"专家人设"，让用户通过内容感知到"这个作者真的懂量化"，信任自然迁移到产品。

### 矛盾 3：个人精力有限 vs 渠道众多

一个人同时做开发+运营，时间是最稀缺资源。撒胡椒面式地在 10 个平台发内容，不如在 2-3 个核心渠道做深。

**解法**：渠道分级，P0 渠道投入 70% 精力，其余维持存在感即可。

## 二、策略定位：开源漏斗 + 内容获客 + 长线变现

### 核心逻辑

```
不是"推广产品"，而是"输出价值，顺便让人知道产品"

技术内容（知乎/掘金/B站）
        ↓ 吸引
量化学习者（流量）
        ↓ 引导
GitHub 开源项目（信任建立）
        ↓ 转化
社群成员（微信群/知识星球）
        ↓ 深度使用
活跃用户（跑通项目、提反馈）
        ↓ 变现（6-12个月后）
付费用户（托管服务/增值功能/课程）
```

### 三个策略原则

1. **内容先行，产品后行**：每一篇技术文章都是一个"免费业务员"，24 小时工作，不用发工资
2. **开源即营销**：GitHub 仓库本身就是最好的落地页，README 写好 = 转化率翻倍
3. **社群是蓄水池**：流量来了留不住等于白来，社群把"路过的人"变成"认识的人"

| **知乎** | 4 | 5(免费) | 5 | 5 | 3 | 22 | P0 |
| **掘金** | 4 | 5(免费) | 4 | 4 | 3 | 20 | P0 |
| **微信公众号** | 3 | 5(免费) | 4 | 3 | 3 | 18 | P1 |

### P0 渠道（投入 70% 精力）

#### 1. GitHub — 开源项目即落地页

**为什么是 P0**：量化开发者的"默认搜索引擎"。一个人搜"quant factor backtest"，第一反应是去 GitHub 搜，不是去百度。

**已具备基础**：MIT 开源、README 已写得相当完整（功能矩阵、架构图、快速启动都有）

**优化清单**：
- [ ] 补充 `CONTRIBUTING.md`：降低贡献门槛，让别人知道怎么参与
- [ ] 补充 `screenshots/` 目录：放 5-8 张前端截图（Dashboard、因子管理、回测结果），README 里嵌入
- [ ] 补充 `docs/quickstart.md`：5 分钟跑通的最短路径（比现有 README 更精简）
- [ ] GitHub Topics 标签：`quantitative-finance` `stock` `factor-model` `backtesting` `a-share` `python` `spring-boot`
- [ ] GitHub Description：一行话说清楚价值（"A股量化投研一体化平台：35+因子×14策略×P1-P6推荐管线×AI分析"）
- [ ] Release v1.0：打一个正式 tag，写好 Release Notes，比"持续开发中"更有可信度
- [ ] Issues 模板：bug report / feature request / question 三种模板

#### 2. 知乎 — 量化技术内容的"主阵地"

**为什么是 P0**：知乎是量化学习者搜索"因子分析""量化入门""Python 量化"的第一站。长尾效应极强——一篇好文章 2 年后还在带流量。

**账号定位**：不做"营销号"，做"量化技术博主"。人设 = 懂代码 + 懂金融 + 愿意分享的独立开发者。

**内容策略**：
- **长文为主**（3000-5000 字），每篇讲透一个技术点
- **标题公式**：`[技术名词] + [实战/详解/从零实现]`，例如"因子 IC 分析实战：从计算到衰减检测"
- **文末固定引导**：GitHub 链接 + 微信群入口（不要硬广，用"完整代码已开源，传送门→"这种自然方式）
- **专栏**：创建"量化投研实战"专栏，系列文章聚合在专栏里

#### 3. 掘金 — 开发者密度最高的技术社区

**为什么是 P0**：掘金用户偏前端/后端开发者，量化平台的"技术架构"内容在这里比知乎更受欢迎。

**差异化策略**：
- 知乎发"量化业务深度"（因子分析、策略设计）
- 掘金发"技术架构深度"（Spring Boot+ClickHouse 怎么搭、因子引擎怎么设计、P1-P6 管线架构）
- 同一个主题，两个平台写不同角度，不重复发

### P1 渠道（投入 20% 精力）

#### 4. 微信公众号 — 私域蓄水池

**定位**：不是获客渠道，是"留存渠道"。知乎/掘金来的流量，用公众号接住。

**运营策略**：
- 每篇知乎/掘金文章同步发公众号（可延后 1-2 天，给平台首发流量）
- 公众号独享内容：每周一篇"量化周报"（本周推荐表现、因子健康度、市场 Regime），展示平台实际运行效果
- 文末固定引导：加微信群入口

#### 5. B站 — 视频内容降门槛

**定位**：文字内容的"入门引流"版。很多学习者看不懂技术文章，但看得懂视频。

**内容策略**：
- 不做"营销视频"，做"操作演示视频"：5-10 分钟录屏，演示平台怎么用
- 选题：从0启动平台、跑一个因子分析、回测一个策略、AI分析个股
- 每月 2-4 条即可，视频制作成本高，不追求量
- 视频简介放 GitHub 链接

#### 6. 聚宽/米筐社区 — 精准但需克制

**定位**：竞品社区，直接发广告会被删。但这里受众最精准。

**策略**：
- 不发广告，发"技术讨论帖"：分享因子分析心得、回测方法论
- 个人签名/主页可放 GitHub 链接
- 重点是建立"专家人设"，让人主动想了解你的项目

### P2/P3 渠道（维持存在感即可）

- **知识星球**：等社群到 200 人再考虑，初期用免费微信群
- **CSDN**：自动同步知乎/掘金内容即可，不用主动运营
- **小红书/抖音**：量化受众不在这里，投入产出比低，暂不做

## 五、转化漏斗设计

### 漏斗全景

```
        曝光层          行动层           信任层           变现层
        ─────          ─────           ─────           ─────
                                        
    知乎文章阅读     →   GitHub 访问    →   跑通项目     →   付费用户
    掘金文章阅读     →   star/clone     →   加微信群     →   
    B站视频播放      →   公众号关注     →   活跃互动     →   
    搜索引擎流量      →   群成员         →   传播推荐     →   
    
    预估转化率：        5-15%             10-20%          1-3%（12个月后）
```

### 各层关键设计

#### 曝光层 → 行动层（转化率 5-15%）

**关键动作**：每篇内容末尾的自然引导

```
✅ 好的引导方式：
"本文涉及的完整代码已开源，包含 35+ 因子和 14 套策略，传送门→ GitHub链接"

❌ 差的引导方式：
"快来 star 我的量化平台项目！！！功能超多！！！"
```

**合规注意**：不能写"用这个平台帮你赚钱"，要写"用于量化学习与研究"

#### 行动层 → 信任层（转化率 10-20%）

**关键动作**：
1. GitHub README 足够吸引人（截图+快速启动+功能矩阵）→ 降低"跑了就关"率
2. 微信群提供真实价值（每周量化周报、问题答疑）→ 把"路人"变"熟人"
3. 有 issue 必回 → 展示项目活跃度

#### 信任层 → 变现层（12 个月后，转化率 1-3%）

**前提**：先想清楚学习者为什么付费。以下是几个可测试的增值方向：

| 增值方向 | 价值主张 | 适合学习者？ | 实现难度 |
|---------|---------|------------|---------|
| **托管 SaaS 版** | 不用自己搭环境，在线使用平台 | 高 | 高（需服务器成本） |
| **数据增值** | Choice 专业数据源接入 | 中（学习者不敏感） | 中 |
| **课程** | 系统性量化入门课程 | 高 | 中 |
| **1v1 咨询** | 量化学习路线、策略定制 | 中 | 低 |
| **策略商城** | 优质策略/因子包 | 低（学习者更想自己写） | 中 |

**建议**：第 3 个月做一次社群调研，问"如果付费，你愿意为什么付费"，用真实数据验证方向，不要自己猜。

## 七、合规红线清单

金融领域推广，以下红线绝对不碰：

### 广告法相关

| 禁用表达 | 替代表达 |
|---------|---------|
| "最好的量化平台" | "功能较完整的开源量化平台" |
| "第一/首家" | "较早开源的 A 股量化平台之一" |
| "稳赚/保证收益" | "历史回测结果，不代表未来表现" |
| "国家级/权威认证" | 不使用此类表述 |
| "限时免费/马上抢" | "项目已开源，欢迎参与" |

### 证券投顾相关

- **绝不荐股**：不推荐具体股票买卖，推荐结果只说"系统产出"，加"不构成投资建议"
- **不承诺收益**：回测结果必须标注"历史模拟，不代表未来"
- **不暗示投顾资质**：平台定位是"量化研究工具"，不是"投资顾问"
- **免责声明**：所有对外内容（文章/视频/README）末尾必须有"仅供学习研究，不构成投资建议"

### 内容合规

- 展示推荐/回测结果时，必须标注"历史数据模拟"
- 不使用"暴利""翻倍""财富自由"等诱导性词汇
- 不蹭"牛市来了""妖股"等热点做营销

### 推荐文末免责声明模板

```
⚠️ 免责声明：本平台仅供量化投资研究与学习使用，不构成任何投资建议。
所有回测结果均为历史数据模拟，不代表未来表现。投资有风险，入市需谨慎。
```

## 九、内容创作效率技巧

### 一鱼多吃

一篇核心内容，产出多个版本：

```
一篇深度技术文章（5000字）
    ↓ 拆解
知乎长文（5000字，业务+技术深度）
    ↓ 改写
掘金文章（3000字，偏技术架构）
    ↓ 精简
公众号推文（1500字，核心要点+周报）
    ↓ 录屏
B站视频（5-10分钟，操作演示）
    ↓ 摘要
朋友圈/群消息（200字，精华预告）
```

### 素材积累习惯

- 每次开发新功能 → 顺手截图 + 记录设计思路 → 攒成文章素材
- 每次踩坑/修 bug → 记录问题和解决方案 → 攒成"踩坑系列"
- 每周推荐结果 → 截图存档 → 攒成"量化周报"素材

### 标题公式库

| 类型 | 公式 | 示例 |
|------|------|------|

| 清单型 | 《[数字]个[领域]必知概念》 | 《量化入门：10 个必须理解的因子指标》 |
