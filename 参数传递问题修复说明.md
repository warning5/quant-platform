# 因子相关性分析API参数传递问题修复

## 问题描述

在调用因子相关性分析API时，后端报错：
```
org.springframework.web.bind.MissingServletRequestParameterException:
Required request parameter 'factorCodes' for method parameter type List is not present
```

## 问题原因

Spring Boot 的 `@RequestParam List<String>` 默认期望的请求参数格式是：
```
factorCodes=value1&factorCodes=value2&factorCodes=value3
```

但是 axios 的默认参数序列化方式可能会将数组参数转换为：
```
factorCodes[]=value1&factorCodes[]=value2&factorCodes[]=value3
```
或者
```
factorCodes=value1,value2,value3
```

这两种格式都不能被 Spring Boot 正确解析为 List<String>，导致参数无法识别。

## 解决方案

### 方案1：前端修改（已采用）

修改 `frontend/src/api/index.js` 中的 `computeCorrelation` 方法，手动构建URL参数：

```javascript
computeCorrelation: (params) => {
  // 手动构建URL参数，确保factorCodes参数正确传递
  const queryParams = new URLSearchParams();
  params.factorCodes.forEach(code => queryParams.append('factorCodes', code));
  queryParams.append('startDate', params.startDate);
  queryParams.append('endDate', params.endDate);
  return api.get(`/factors/correlation?${queryParams.toString()}`);
},
```

**优点**：
- 前端完全控制参数格式
- 不依赖后端的参数解析方式
- 兼容性更好

**效果**：
生成的URL参数格式为：
```
/factors/correlation?factorCodes=MOM20&factorCodes=MOM60&startDate=2023-01-01&endDate=2024-12-31
```

### 方案2：后端修改（已采用）

修改 `backend/src/main/java/com/quant/platform/factor/controller/FactorController.java`，添加参数验证和更好的错误提示：

```java
@GetMapping("/correlation")
@Operation(summary = "计算因子相关性矩阵")
public ApiResponse<List<Map<String, Object>>> computeCorrelation(
        @RequestParam(required = false) List<String> factorCodes,
        @RequestParam(required = false) @DateTimeFormat(iso = DateTimeFormat.ISO.DATE) LocalDate startDate,
        @RequestParam(required = false) @DateTimeFormat(iso = DateTimeFormat.ISO.DATE) LocalDate endDate) {
    if (factorCodes == null || factorCodes.isEmpty()) {
        throw new IllegalArgumentException("factorCodes参数不能为空");
    }
    if (startDate == null || endDate == null) {
        throw new IllegalArgumentException("startDate和endDate参数不能为空");
    }
    return ApiResponse.success(correlationService.computeCorrelationMatrix(factorCodes, startDate, endDate));
}
```

**优点**：
- 添加了参数验证，提供更友好的错误提示
- 将参数设置为 `required = false`，然后在方法内部验证，避免框架抛出异常
- 错误信息更清晰，便于调试

### 方案3：后端使用@RequestParam的name属性（未采用，供参考）

```java
@RequestParam(name = "factorCodes") List<String> factorCodes
```

这种方式可以明确指定参数名称，但仍然需要前端配合传递正确的格式。

### 方案4：后端使用@ModelAttribute或DTO（未采用，供参考）

创建一个DTO类来接收所有参数：

```java
public class CorrelationRequest {
    private List<String> factorCodes;
    private LocalDate startDate;
    private LocalDate endDate;
    // getters and setters
}

@GetMapping("/correlation")
public ApiResponse<List<Map<String, Object>>> computeCorrelation(
        @ModelAttribute CorrelationRequest request) {
    // ...
}
```

这种方式更灵活，但对于简单的GET请求来说可能有些过度设计。

## URL参数格式示例

修复后的URL格式：
```
GET /api/factors/correlation?factorCodes=MOM20&factorCodes=MOM60&factorCodes=VOL20&startDate=2023-01-01&endDate=2024-12-31
```

Spring Boot会自动将多个 `factorCodes` 参数解析为 `List<String>`：
```java
List<String> factorCodes = ["MOM20", "MOM60", "VOL20"];
```

## 注意事项

1. **URLSearchParams兼容性**：URLSearchParams 是现代浏览器支持的标准API，在所有主流浏览器中都能正常工作
2. **参数顺序**：参数顺序不影响后端解析
3. **参数编码**：URLSearchParams 会自动处理特殊字符的URL编码
4. **重复参数名**：使用 `append()` 方法可以正确处理重复的参数名

## 相关知识

### Spring Boot参数绑定规则

Spring Boot 的 `@RequestParam` 支持以下格式：

1. **简单类型**：
   ```
   ?name=value
   -> String name = "value"
   ```

2. **数组类型**：
   ```
   ?names=value1&names=value2
   -> List<String> names = ["value1", "value2"]
   ```

3. **日期类型**：
   ```
   ?date=2023-01-01
   -> LocalDate date = LocalDate.parse("2023-01-01")
   ```

### Axios参数序列化

Axios 默认的参数序列化配置：
- 数组使用 `indices` 格式：`a[0]=v1&a[1]=v2`
- 可以通过配置修改：`paramsSerializer: params => qs.stringify(params, { arrayFormat: 'repeat' })`

## 总结

通过前端手动构建URL参数和后端添加参数验证，成功解决了因子相关性分析API的参数传递问题。这种前后端配合的方式既保证了参数格式的正确性，又提供了友好的错误提示。
