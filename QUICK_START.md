### 4. 多策略对比

#### 前提条件
- 系统中至少存在 **2 个已完成（COMPLETED）状态**的回测任务

#### 操作步骤
1. 点击左侧菜单 **回测管理 > 策略对比**
2. 在「选择回测任务」下拉框中，选择 **2~8 个**已完成的回测（支持搜索）
3. 点击「开始对比」按钮
4. 逐一查看以下结果：
   - **净值曲线对比图**：观察各策略在相同时间段内的涨跌幅走势
   - **综合能力雷达图**：快速判断哪个策略在六维指标上更均衡
   - **核心指标速览**：一眼看出年化收益、回撤、Sharpe 的相对表现
   - **详细指标表**：向右滑动查看全部 13 列指标，首行为年化收益最高的策略

#### 读图要点
- 净值曲线图纵轴为**涨跌幅（%）**，非绝对净值，便于跨时间段横向对比
- 雷达图越接近外圈（面积越大）表示该维度表现越好
- 表格中红色表示正向指标表现好，绿色表示负向指标表现好

### 5. 参数优化（网格搜索）

#### 操作步骤
1. 点击左侧菜单 **回测管理 > 参数优化**
2. 在左侧配置区填写：
   - **策略**：选择要优化的策略
   - **开始/结束日期**：建议选取 1~3 年的历史数据
   - **目标函数**：推荐选「最大化 Sharpe 比率」（风险调整后收益）
   - **并行度**：建议设 2~3，避免过多占用系统资源
3. 在「参数网格」区域：
   - 点击「**+ 添加参数**」
   - 选择参数名（如「持仓数量」），设置类型为「范围」
   - 填写最小值、最大值、步长（系统自动展示参数值数量）
   - 可添加多个参数，组合数 = 各参数值数量之积
4. 注意底部「预计运行 N 次回测」的提示，建议控制在 **50 次以内**
5. 点击「**开始优化**」，右侧实时显示进度
6. 完成后：
   - 查看**最优参数组合卡片**（金色边框）获取建议参数
   - 查看**热力图**了解各参数组合对目标函数的影响规律
   - 在**结果表格**中对比所有参数组合的绩效

#### 参数配置技巧
- **范围模式**：适合连续数值参数，如持仓数 10~30 步长 5，共 5 个值
- **列表模式**：适合离散值，如直接输入 `10,20,30`（逗号分隔）
- 建议先粗粒度扫描（步长大），再在最优区间附近细化搜索

### 6. 蒙特卡洛模拟

#### 操作步骤
1. 进入任意**已完成**回测的详情页
2. 找到「**蒙特卡洛模拟**」Tab（位于报告页面的标签页中）
3. 在卡片右上角设置：
   - **模拟次数**：200（快速）/ 500（默认）/ 1000（精确）
   - **预测期**：1季度 / 半年 / 1年（默认）/ 2年
4. 点击「**运行模拟**」，等待数秒（500次约 2~5 秒）
5. 依次查看：
   - **6 个核心指标卡片**：重点关注正收益概率和 VaR/CVaR
   - **净值置信区间图**：P50（蓝色）为中位数预期，P5（红虚）为悲观情形
   - **年化收益率分布直方图**：分布越集中于正值，策略越稳健
   - **最大回撤分布直方图**：分布越集中于小值，回撤风险越可控

#### 如何解读结果
| 指标 | 含义 | 参考标准 |
|------|------|----------|
| 正收益概率 > 60% | 多数情形下策略盈利 | 绿色标注 |
| VaR（95%）< 20% | 极端情形下损失可控 | 越低越好 |
| CVaR（95%）| 超出 VaR 时的平均损失 | 评估尾部风险 |
| P50 年化收益 | 最可能实现的年化收益率 | 与历史对比 |

### 7. 因子权重优化

#### 操作步骤
1. 点击左侧菜单 **因子管理 > 权重优化**
2. 在顶部控制区填写：
   - **因子**：在多选框中选择 **2~10 个**因子（支持搜索因子代码/名称）
   - **优化方法**：
     - 「等权」— 最简单，无偏估计
     - 「Markowitz」— 最大化历史夏普，**推荐用于追求高收益调整**
     - 「风险平价」— 均衡分散风险，**推荐用于追求稳健性**
   - **开始/结束日期**：建议覆盖 1~2 年的因子历史数据
3. 点击「**开始优化**」
4. 查看结果：
   - **4 个指标卡**：查看组合预期年化收益、波动率和夏普比率
   - **权重饼图**：了解建议的权重分配
   - **权重明细表**：查看每个因子的具体权重和历史表现
   - **相关系数热力图**：颜色越绿表示因子间越独立（分散效果好）
   - **有效前沿图**（仅 Markowitz）：横轴波动率、纵轴收益率，颜色越绿 Sharpe 越高

#### 最佳实践
1. 先用「**因子相关性分析**」（因子管理 > 因子相关性）筛选出相关性低（< 0.3）的因子组合
2. 再用「权重优化」对筛选后的因子进行权重配置
3. 对比「Markowitz」和「风险平价」两种方法的结果，选择与自身风险偏好匹配的方案
4. 将建议权重用于「新建回测」中的因子权重配置

## 推荐分析流程

```
1. 因子相关性分析  →  筛选低相关因子组合（< 0.3）
2. 因子衰减分析    →  评估各因子有效期，选择稳健因子
3. 因子权重优化    →  计算最优权重（推荐 Markowitz 或风险平价）
4. 创建并运行回测  →  使用优化后的权重配置
5. 参数优化        →  对止损/持仓等参数进行网格搜索微调
6. 多策略对比      →  横向对比不同权重方案和参数组合的表现
7. 蒙特卡洛模拟    →  评估最优策略的样本外鲁棒性和尾部风险
```

## v1.1.0 功能快速上手

### 1. 因子衰减分析(因子有效期)

#### 如何查看
1. 进入**因子管理 > 因子列表**
2. 点击任意因子进入**因子详情**
3. 在**因子检测**标签页,点击**新增检测**
4. 等待检测完成后,向下滚动查看**因子衰减分析**卡片

#### 理解指标
- **因子有效期**: 因子预测能力能持续多少期,越长越好
- **半衰期**: IC降到初始值一半的期数,越小衰减越快
- **衰减系数**: 拟合指数衰减的系数,值越大衰减越快
- **拟合优度R²**: 模型拟合程度,越接近1越可信

#### 使用场景
- 选择有效期长的因子进行长期投资
- 根据衰减速度调整调仓频率
- 评估因子在持有期的稳定性

### 2. 因子相关性分析

#### 如何使用
1. 点击左侧菜单**因子管理 > 因子相关性**
2. 在因子选择下拉框中选择要分析的因子(至少2个)
3. 选择日期范围
4. 点击**计算相关性**
5. 查看热力图和详细数据表

#### 理解热力图
- **红色区域**: 强正相关(>0.7),因子重叠严重
- **蓝色区域**: 强负相关(<-0.7),因子对立
- **白色区域**: 无相关,因子独立
- **对角线**: 永远为1(自身与自身)

#### 使用建议
- 避免选择相关性>0.7的因子组合
- 优先选择相关性<0.2的因子组合
- 不同类型的因子(动量+价值)通常相关性低

### 3. MySQL数据库配置

#### 切换到MySQL

**步骤1: 安装MySQL**
- Windows: 下载MySQL Installer
- Mac: `brew install mysql`
- Linux: `sudo apt-get install mysql-server`

**步骤2: 创建数据库**
```bash
mysql -u root -p
CREATE DATABASE quantdb CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
EXIT;
```

**步骤3: 修改配置**
编辑 `backend/src/main/resources/application-mysql.yml`:
```yaml
spring:
  datasource:
    username: root
    password: 你的MySQL密码
```

**步骤4: 启动应用**
```bash
cd backend
mvn spring-boot:run -Dspring-boot.run.profiles=mysql
```

#### 验证MySQL连接
启动后查看日志,应该看到:
```
HikariPool-1 - Starting...
HikariPool-1 - Start completed.
```

## 常见问题

### Q1: 因子衰减分析图表不显示?
**A**: 确保因子检测已完成,并且IC序列数据存在。旧版本的检测报告需要重新运行检测。

### Q2: 因子相关性分析计算很慢?
**A**: 这是正常的,相关性分析需要遍历所有日期和因子组合。可以通过缩短日期范围或减少因子数量来加速。

### Q3: MySQL连接失败?
**A**: 检查以下几点:
- MySQL服务是否启动
- 用户名密码是否正确
- 数据库quantdb是否存在
- 防火墙是否允许3306端口

### Q4: 切换数据库后数据丢失?
**A**: H2和MySQL的数据是独立的。如果需要迁移数据，需要手动导出导入。

### Q5: 参数优化长时间不结束?
**A**: 参数组合数过多时耗时会很长。建议：
- 减少参数数量或缩小取值范围
- 降低并行度避免内存压力
- 缩短回测日期范围

### Q6: 蒙特卡洛报错「历史数据不足」?
**A**: 需要至少 20 个交易日的净值数据。确认该回测的时间跨度不低于 1 个月。

### Q7: 因子权重优化报错「有效数据点不足」?
**A**: 所选因子在指定日期范围内的公共交易日不足 10 个。请：
- 扩大日期范围（建议至少 1 年）
- 确认所选因子已完成批量计算，factor_value 表中有数据

### Q8: 多策略对比页面选不到回测？
**A**: 只能选择状态为 COMPLETED 的回测任务。如果列表为空，请先完成至少 2 个回测。

## 最佳实践

### 因子组合选择
1. 使用**因子相关性分析**找出相关性低的因子
2. 使用**因子衰减分析**评估因子的持续性
3. 结合**IC分析**和**分组测试**选择高表现因子
4. 建议选择3-5个因子组合,避免过度拟合

### 数据库选择
- **开发测试**: 使用H2(无需安装,开箱即用)
- **生产环境**: 使用MySQL(性能好,支持高并发)
- **数据迁移**: 使用mysqldump导入导出

### 性能优化
- 因子计算时选择合理的时间区间
- 因子检测时选择适当的股票池
- 定期清理旧的检测报告
- MySQL配置innodb_buffer_pool_size为可用内存的70%
