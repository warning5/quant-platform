## 第一部分：问题1 — 手动操作全流程

### 一、总览：从因子到复盘的完整链路


### 二、每一步操作详解

#### Step 1：因子配置（筛选"优质+低价"的因子组合）

**目标**：配置一组能同时反映"公司质地好"和"价格相对低"的因子

| OPER_CASHFLOW_RATIO | 财务因子 | ✅ 已有 | 现金流健康 |
| GROSS_PROFIT_MARGIN | 财务因子 | ✅ 已有 | 毛利率竞争力 |
| MOM20/MOM60 | 技术因子 | ✅ 已有 | 动量/回撤 |

**需补充**：
| 因子 | 说明 | 开发量 |
|------|------|--------|

**操作方式**：
1. 进入"因子"页面 → 创建自定义因子
2. 为新因子编写计算脚本（Python，复用现有因子计算框架）
3. 触发计算，验证IC/IR

**推荐因子权重方案（"低价优质"策略）**：

```
质量维度(50%): ROE(15%) + NET_PROFIT_GROWTH(15%) + OPER_CASHFLOW_RATIO(10%) + DEBT_RATIO(10%)
估值维度(35%): PE_PERCENTILE(15%) + PB_PERCENTILE(10%) + DIVIDEND_YIELD(10%)
技术维度(15%): RSI5(5%) + 缠论买点(5%) + PRICE_52W_HIGH_PCT(5%)
方向: ROE/成长/现金流/股息 → 正向(越大越好)；PE分位/PB分位/负债率/RSI/回撤 → 反向(越小越好)
```

### 二、关键节点详解

#### 节点1：日线判断方向，分钟线判断介入位置

**日线判方向（已有能力）**：

日线数据的用途是回答"这只股票是否处于低位、是否值得关注"：

| 判断项 | 日线指标 | 逻辑 |
|--------|---------|------|
| 趋势方向 | MA20/MA60排列 | 均线空头→下行趋势，多头→上行趋势 |
| 是否低位 | PE/PB 3年分位 < 30% | 估值处于历史低位区间 |
| 回撤幅度 | 距52周高点回撤 > 20% | 价格大幅回落 |
| 底部信号 | 缠论底分型 / 一买二买 | 缠论识别到趋势转折 |
| 量价配合 | 缩量横盘后放量 | 底部吸筹特征 |
| 动量反转 | RSI < 30 或 MOM20拐头 | 超卖后动能恢复 |

**以上全部基于日线，系统已具备。**

**分钟线判介入位置（需新增）**：

分钟线的用途是回答"今天/明天什么价位可以买"：

| 判断项 | 分钟线指标 | 逻辑 |
|--------|----------|------|
| 精确支撑位 | 30分钟/60分钟缠论笔 | 日线"大概在底部"→分钟线"精确支撑在 X 元" |
| 分时买点 | 30分钟二买确认 | 日线出现底分型后，等30分钟级别二买确认 |
| 入场区间 | 分钟中枢下沿 ± 1% | 在中枢下方入场，中枢内部观望 |
| 当日时机 | 分时量价突破 | 开盘30分钟放量突破前日收盘价 → 当日可入 |
| 避免追高 | 分钟级别顶分型 | 出现顶分型 → 不追 |

**具体操作逻辑**：
```
日线判断: 缠论一买出现 + PE分位<30% + ROE>10%  →  "方向确认，值得关注"
    ↓
拉取30分钟K线（近20个交易日）
    ↓
分钟线判断:
  - 当前30分钟级别是否有底分型？ → 是：等待二买确认
  - 中枢下沿价格？ → 买入价 = 中枢下沿 × 0.99
  - 止损价 = 中枢下沿 × 0.95（约2%止损）
    ↓
输出: 买入价区间 [中枢下沿×0.99, 中枢下沿×1.01]
```

**分钟线数据采集方案**：
- **不是全市场采集**，只对规则引擎筛出的30只候选股拉取
- 数据源：腾讯证券 `proxy.finance.qq.com` 接口（与北交所日线接口同源，支持 `1m/5m/15m/30m/60m` 频率）
- 每只股票请求1次（拉近20交易日30分钟线 ≈ 960条），30只共30次请求
- 采集时间：每日 07:30 执行，耗时约1分钟

#### 节点2：为什么需要实时推送？腾讯实时接口可用性

**为什么需要实时推送**：

场景：系统在早上8:30给出推荐——"XX股份，买入价 15.2~15.5 元"。
到了盘中，这只股票可能在10:15才跌到 15.3 元。

**没有实时推送**：你不知道价格到了，可能错过最佳买点
**有实时推送**：10:15 价格触达 15.3 → 即时提醒"目标价到达，可介入"

| 场景 | 无推送 | 有推送 |
|------|--------|--------|
| 买入价到达 | 不知道，可能错过 | 即时提醒 |
| 止损触发 | 晚发现，损失扩大 | 即时提醒 |
| 盘中突发利好 | 不知道 | 新闻信号触发提醒 |

**腾讯实时接口可用性评估**：

系统已经在用腾讯的两个接口：

| 接口 | URL | 用途 | 稳定性 | 限流 |
|------|-----|------|--------|------|
| 实时行情快照 | `http://qt.gtimg.cn/q=sh600519` | 内外盘数据(已有) | ⭐⭐⭐⭐ 稳定运行数月 | 批量查询约500只/次，间隔0.5秒 |
| 历史K线 | `https://proxy.finance.qq.com/ifzqgtimg/appstock/app/newfqkline/get` | 北交所日线(已有) | ⭐⭐⭐⭐ 稳定 | 单次640条，间隔0.3秒 |

**扩展到盘中监控的可行性**：

```
盘中监控流程（09:30~15:00）：
  每60秒 → 批量查询候选池股票实时价格（qt.gtimg.cn）
  → 与目标买入价比较

```

| 评估项 | 结论 |
|--------|------|
| 接口可用 | ✅ 已在用，无需新接口 |
| 限流风险 | 30只候选 × 每60秒1次 = 0.5 QPS，远低于限流阈值 |

**理由**：

1. **中文金融理解能力顶级**：DeepSeek在中文金融文本上的训练质量公认第一梯队，理解"底分型""中枢""估值分位"等术语零障碍

3. **1M上下文窗口**：一只股票的全量分析数据（日线+财务+缠论+新闻+研报）约3k tokens，30只合计90k，轻松装下
4. **国内直连**：api.deepseek.com 无需代理，延迟低
5. **Thinking模式**：Pro版本支持深度推理模式，适合生成"为什么推荐"的逻辑链
6. **缓存命中**：DeepSeek Flash缓存命中输入仅$0.0028/M，每日Prompt模板相同，大部分输入可命中缓存

## 第三部分：两个问题的关系

### 一、核心论断：递进 + 复用关系

**问题1（手动流程）是问题2（全自动方案）的基础和子集。**

```
问题1: 人操作 → 平台辅助 → 人决策
问题2: 平台全自动 → 人只看结果

两者不是平行方案，而是：
  问题1的每一步 → 问题2自动化该步骤 → 叠加大模型推理 → 输出可操作结果
```

**递进关系**：

| 问题1的步骤 | 问题2的自动化升级 | 新增了什么 |
|------------|-------------------|-----------|
| 手动配置因子 | 因子自动计算（已有） | 无，直接复用 |
| 手动定义策略 | 策略预配置+定时触发 | 定时任务调度 |
| 手动点"生成推荐" | 每日07:15自动执行 | Cron调度 |
| 手动逐只看分析 | 大模型批量推理 | **LLM推理层** |
| 手动判断买价 | 分钟线+缠论→买价区间 | **分钟级分析** |
| 手动盯盘 | 实时价格监控+推送 | **监控循环+推送** |
| 手动记录持仓 | 系统自动记录 | **仓位管理** |
| 手动复盘 | 自动生成每日报告 | **报告模板** |

## 附录：腾讯实时接口技术细节

### qt.gtimg.cn 实时行情快照

```
URL: http://qt.gtimg.cn/q=sh600519,sz000001
返回格式: v_sh600519="1~贵州茅台~600519~1320.92~1325.00~1321.00~..."
字段顺序(用~分隔):
  [3]当前价 [4]昨收 [5]今开 [6]成交量 [7]外盘 [8]内盘
  [9]买一价 [10]买一量 ... [19]卖一价 [20]卖一量
  [30]换手率 [31]PE [32]振幅 [38]总市值 [39]流通市值
  [43]PB [44]涨停价 [45]跌停价 [46]量比

批量限制: 约500只/次
频率: 0.5秒间隔即可稳定
注意事项: 用HTTP而非HTTPS（Windows下HTTPS握手慢7秒）
```

### proxy.finance.qq.com 分钟K线

```
URL: https://proxy.finance.qq.com/ifzqgtimg/appstock/app/newfqkline/get
参数: param=sh600519,m30,2026-06-01,2026-06-12,640,qfq
频率: 1m/5m/15m/30m/60m
限制: 单次640条
延迟: ~0.3秒/次
```

### 接口稳定性评估

- 已持续使用3个月+，未出现封禁或限流
- 双数据源热备（腾讯+AKShare/东财）已有先例（内外盘脚本中实现）
- 盘中监控仅30只/分钟级轮询，压力极小
