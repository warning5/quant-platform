## 问题2：入场时机判断模块实现 ✅

### 新增文件
**EntrySignalAnalyzer.java**（~360行）
- 4维信号评分：突破确认(35) + 均线排列(20) + 量价配合(25) + 回踩确认(20) = 100
- `analyze()`：主入口，拉取 m5 K线 → 4维评分 → 返回 BreakoutSignal
- `fallbackPriceOnly()`：K线获取失败时的纯价格降级判断（public，供Monitor调用）
- `fetchM5Kline()`：调用腾讯 proxy.finance.qq.com 拉取最近30根m5 K线，3s超时
- `guessTencentCode()`：根据代码前缀自动判断 sh/sz/bj

### 修改文件
**IntradayMonitorService.java**（重构）
- 推送去重：从 `Set<String>` 改为 `Map<String, LocalDateTime>`（记录推送时间）
- 新增 `canPush()`/`markPushed()`：按信号类型独立冷却期
- 冷却期配置：`buy-cooldown-minutes: 30`（买入），`stop-cooldown-minutes: 60`（止损）
- **止损独立判断**：`currentPrice <= stopLoss` 直接推送，不参与评分，且 `continue` 跳过买入判断
- **竞价阶段跳过K线分析**：`inAuction` 判断，直接用 `fallbackPriceOnly` 降级
- 触发区间外扩：`proximity-pct: 0.02`（买入区间上下各2%）

### 配置项（application.yml 可覆盖）
```yaml
quant:
  monitor:
    signal:
      threshold: 70        # 评分阈值，>=70推送强烈买入
      proximity-pct: 0.02  # 触发区间外扩比例
    cooldown:
      buy-minutes: 30      # 买入信号冷却期
      stop-minutes: 60     # 止损信号冷却期
```

## 监控数据源双源合并 ✅

### 问题
盘中监控只读 `llm_analysis` 表（BUY推荐），但 LLM 分析可能全是 SKIP/WATCH，导致监控0股票。
而 `stock_recommendation`（智能推荐）每天都有 actionTag=BUY 的推荐结果，却没接入监控。

### 改造
`loadTargetPrices()` 改为双源合并：
1. **数据源1（LLM）**：`llm_analysis WHERE recommendation='BUY'` — 精确止损/目标价
2. **数据源2（推荐）**：`stock_recommendation WHERE action_tag='BUY'` — 用 suggestedBuyPrice 换算
   - buyPriceLow = suggestedBuyPrice × 0.95
   - buyPriceHigh = suggestedBuyPrice × 1.05
   - stopLoss = suggestedBuyPrice × 0.92
   - targetPrice = closePrice × 1.20
3. **去重规则**：LLM 优先，已有则跳过推荐源

**TargetPriceInfo 新增 `source` 字段**（"LLM"/"推荐"），前端表格增加来源列。
