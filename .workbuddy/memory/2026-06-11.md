### 方案C：策略置信度

**数据模型**: `strategy_confidence` 表
- 字段: strategy_id, level(HIGH/NORMAL/LOW/SUSPENDED/UNTRAINED), score(0~100)
- 维度得分: hit_rate_score(0~40) + return_score(0~25) + drawdown_score(0~20) + volatility_score(0~15)
- 元信息: sample_size, data_as_of_date
- 索引: `uk_strategy_date(strategy_id, data_as_of_date)`

**置信度算法** (满分100分):
1. **命中率维度**(40分): 线性映射 `hitRate × 40`
2. **平均收益维度**(25分): `[-3%, +2%] → [0, 25]`
3. **最大回撤维度**(20分): `[0%, -10%] → [20, 0]`
4. **波动率维度**(15分): `[1.5%, 6%] → [15, 0]`

**等级**: ≥70=HIGH, 50~69=NORMAL, 30~49=LOW, <30=SUSPENDED

**后端改动**:
1. `StrategyConfidence.java` (domain) + `StrategyConfidenceMapper.java`
2. `StrategyConfidenceService.java` — 核心逻辑:
   - `calculateAndSave(strategyId)` — 收集近10期数据，计算4维度得分，持久化
   - `getAdjustedTopN(originalTopN, confidence)` — 根据等级调整topN (HIGH→原值, NORMAL→原值, LOW→max(5,topN-8), SUSPENDED→-1)
   - `shouldShowWarning(confidence)` — 判断是否需要显示警告
3. `StrategyConfidenceController.java` — REST API: `GET/POST /strategy-confidence/*`
4. `RecommendationService.java`:
   - 注入 `StrategyConfidenceService`
   - **Step 0.5**: 在上期命中率检查之后、Step1之前，检查置信度并调整topN
   - **追踪后**: 在黑名单评估之后，调用 `calculateAndSave()` 更新置信度

**前端改动**:
1. `api/index.js` — 新增 `confidenceApi`
2. `RecommendationList.jsx`:
   - 导入 `confidenceApi`, `CONFIDENCE_CONFIG`
   - state: `confidenceData`, `allConfidence`, `confidenceWarningVisible`
   - **策略下拉**: 每个选项旁显示置信度分数(Tag)和详情图标
   - **推荐列表顶部**: 置信度状态栏(分数/等级/命中率/平均收益 + "查看详情"按钮) + 未训练提示
   - **置信度详情弹窗**: 综合进度条 + 4维度得分明细(Progress) + 建议文本
   - **useEffect**: 策略切换时自动加载置信度；首次加载时获取所有策略置信度
